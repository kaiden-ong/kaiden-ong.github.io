Schedule Your Vaccine

// built with Java & and Azure Database

What Is It?

Amid the peak of the Covid-19 pandemic, I created a vaccine scheduler that could provide useful functionalities to successfully administer vaccine shots. At the time, vaccines were limited and nearly impossible to find, let alone set up an appointment and actually get the shot, so the functions of scheduling, cancelling, and viewing appointments were necessary. Utilizing SQL for database management and Java for implementation, the scheduler presents an approach to seamlessly book appointments, view schedules, and login as a patient or caregiver. Some skills and tools that I used to create this project include:

Microsoft Azure
SQL Queries
Java Programming
Database Management

First Steps

As our final project in CSE 414 (Intro to Database Systems), we were tasked to implement this Covid-19 vaccine scheduler, connected to an Azure SQL Database. Using course content learned throughout the quarter like SQL, database design, relational algebra, transactions and more, our scheduler was made with numerous key functions. The scheduler had two types of users, patients and caregivers, each who had different privileges within the application, and it also keeps track of vaccine doses, increasing & decreasing inventory as needed. An additional function that was designed, but not implemented was managing hospitals and the insurance companies that were accepted at each one. This was one of my first "large" projects, and stands out because of its practicality and relevance even to this day.

Timeline

① Setting Up Database on Azure

The first step was to set up my Azure account and design the relational database to include tables for patients, caregivers, and vaccine doses.

② Writing the Java

Along with managing the database, I needed functions like logging in/out, checking strength of passwords, and correctly modeling the patients, caregivers, and vaccine.

③ Writing/Testing SQL Queries

The Java program would send out queries to the database in response to actions done by the user. These ranged from scheduling appoitments to decreasing vaccine doses.

④ Testing Combined Functionality

After I had implemented all functionalities, it was important to review & ensure read and writes on the database were executing correctly.

Lessons Learned

Prior to completing this project, I was unfamiliar with how databases worked, how to get information and data from a databse, or how to even set one up. So this project taught me a lot about managing information and design within a database, as well as how to write efficient queries. As I navigated through the process of creating this vaccine scheduler, I honed my skills in structuring and fetching data, and they have prepared me for future endeavors in software development.
